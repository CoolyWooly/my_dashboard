# 🐳 Docker Dashboard

Лёгкий веб-дашборд для мониторинга Docker-контейнеров на сервере (Hetzner и любом другом).

## Что умеет

- **Список контейнеров** — статус, образ, аптайм, порты + кнопки start / stop / restart
- **Живые логи** — стриминг логов контейнера в реальном времени (WebSocket)
- **Метрики контейнеров** — CPU, RAM, сетевой трафик по каждому контейнеру
- **Метрики хоста** — CPU, RAM, диск, аптайм сервера

Бэкенд читает данные напрямую из Docker socket через `dockerode`. Без внешних БД и зависимостей.

## Стек

Node.js · Express · dockerode · ws · systeminformation · ванильный JS-фронтенд (без сборки).

## Локальный запуск

```bash
npm install
npm start
# http://localhost:3000
```

> На macOS метрики хоста показывают твой Mac, а контейнеры — из Docker Desktop. На сервере всё будет по-настоящему.

## Запуск на сервере (Docker)

```bash
# на сервере, в папке проекта
docker compose up -d --build
```

Дашборд поднимется на `127.0.0.1:3000` (только локально для безопасности).

### Как открыть в браузере

По умолчанию compose публикует дашборд на порту **9000**: `http://SERVER_IP:9000`.
Дашборд **управляет контейнерами**, поэтому защити его basic-авторизацией (см. ниже) или спрячь за VPN/SSH-туннелем.

**Basic-авторизация** включается заданием `DASH_USER` и `DASH_PASS`. Создай рядом с `docker-compose.yml` файл `.env` (он в `.gitignore`):

```env
DASH_USER=admin
DASH_PASS=придумай-длинный-пароль
```

Без этих переменных дашборд стартует **без** авторизации.

## Конфигурация

| Переменная    | По умолчанию           | Описание                                  |
| ------------- | ---------------------- | ----------------------------------------- |
| `PORT`        | `3000`                 | Порт веб-сервера внутри контейнера        |
| `DASH_USER`   | —                      | Логин basic-авторизации (вкл. вместе с PASS) |
| `DASH_PASS`   | —                      | Пароль basic-авторизации                  |
| `DOCKER_HOST` | `/var/run/docker.sock` | Адрес Docker (для удалённого хоста)       |

## Безопасность

- Доступ к Docker socket = root-доступ к хосту. Не открывай дашборд наружу без `DASH_USER`/`DASH_PASS`.
- `.env` с паролем не коммитится (в `.gitignore`) — лежит только на сервере.
- В `docker-compose.yml` socket монтируется для управления контейнерами. Для режима «только чтение» добавь `:ro` (тогда start/stop работать не будут).

## API

| Метод  | Путь                              | Описание                       |
| ------ | --------------------------------- | ------------------------------ |
| GET    | `/api/containers`                 | Список контейнеров             |
| GET    | `/api/containers/:id`             | Детальная инфо (inspect)       |
| POST   | `/api/containers/:id/:action`     | start \| stop \| restart       |
| GET    | `/api/stats`                      | Метрики всех контейнеров       |
| GET    | `/api/system`                     | Метрики хоста                  |
| WS     | `/api/logs?id=<containerId>`      | Стриминг логов                 |
